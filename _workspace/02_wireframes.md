## 1. 설계 원칙 (3초 규칙)

1. **3초 안에 다음 행동이 보이는가.** 댄스 학습자는 영상을 보러 왔지 UI를 학습하러 온 게 아니다. 모든 화면은 진입 즉시 (1) 무엇을 보는 곳인지 (2) 다음 한 번의 탭이 무엇인지가 시각적으로 명확해야 한다.
2. **썸네일이 주인공.** UI는 무채색 무대(stage) + 단일 꿀 액센트. 콘텐츠 카드가 화면 면적의 다수를 차지한다.
3. **모바일 퍼스트.** 데이터상 P4(숏츠·모바일)가 최대 진입 표면. 모든 페이지는 360px 기준으로 먼저 성립하고 데스크탑으로 확장한다.
4. **핵심 동선은 보편 패턴.** 탐색·필터·시청은 검증된 패턴(그리드/필터바/플레이어). 차별화 인터랙션(숏츠 풀스크린, 히어로 모션, 스텝 타임라인)은 동선을 막지 않는 레이어에 배치.
5. **4상태 필수.** 모든 데이터 영역은 데이터있음 / 빈상태 / 로딩 / 에러 4가지를 정의한다. 빈 상태 설계가 없으면 구현자가 임의 처리해 품질이 갈라진다.

### 반응형 분기점 (전역)

| 분기 | 폭 | 컨테이너/거터 | 그리드 기본 |
|------|----|--------------|------------|
| **모바일** | < 768px | full-bleed, gutter 16px | 영상 1열 / 숏츠 풀스크린 |
| **태블릿** | 768–1023px | gutter ~24px | 영상 2열 / 숏츠 중앙 컬럼 |
| **데스크탑** | ≥ 1024px | max `--content-max`(1280) 중앙, gutter `clamp(16,4vw,40)` | 영상 3–4열 / 숏츠 중앙 9:16 + 사이드 |

토큰: 섹션 상하 패딩 `--section-py-desktop`(96) / `--section-py-mobile`(64).

## 2. 글로벌 셸 (모든 페이지 공통)

### 2.1 데스크탑 — 상단 헤더 (sticky)

```
┌───────────────────────────────────────────────────────────────────────┐
│  꿀초아tv      홈   학습   라이브러리   숏츠   소개          [🔍] [☾/☀]  │  ← sticky, blur(12px)
└───────────────────────────────────────────────────────────────────────┘   stage-950 88%, 하단 stage-700 1px
       ▲로고(font-display)        ▲활성: honey-300 + 하단 2px honey-400 인디케이터
```

- 스크롤 다운 시 높이 72→56px 축소(design-system §5.6), 업 시 복귀.
- 검색 아이콘은 후순위(MVP는 `/library` 내장 검색으로 대체). 아이콘 슬롯만 자리 확보, 클릭 시 `/library`로 포커스 이동.
- 테마 토글: 다크↔라이트(`[data-theme]`).

### 2.2 모바일 — 상단 미니 헤더 + 하단 탭바

```
┌─────────────────────────────┐
│ 꿀초아tv              [🔍][☾]│  ← 상단: 로고 + 검색·테마 (56px)
├─────────────────────────────┤
│                             │
│        (페이지 본문)         │
│                             │
├─────────────────────────────┤
│  [홈]  [학습]  [라이브러리] [숏츠] │  ← 하단 탭바 (1급 항목 4개, 56px+safe-area)
```

- 하단 탭바 항목: **홈 / 학습 / 라이브러리 / 숏츠** (소개·검색은 탭바 제외 — 상단/홈 푸터 경유). 숏츠는 P4 진입 표면이라 1급 고정.

## 3. 공용 컴포넌트 사양 (페이지에서 반복 참조)

### 3.1 영상 카드 (16:9) — `VideoCard`

```
┌──────────────────────────┐
│ [뱃지]            ▶       │  ← 썸네일 16:9, radius-thumb. 좌상단 뱃지 1개
│      (썸네일 이미지)        │     우하단 재생시간 칩(stage-950 70%)
│                    9:04   │
├──────────────────────────┤
│ 제목 두 줄까지 클램프 …      │  ← text-base/700, 2줄
│ 조회 1.2만 · 2026         │  ← text-sm/text-muted, tabular-nums
└──────────────────────────┘
```

| 요소 | 데이터 바인딩 (ContentCard) | 인터랙션 | 상태 |
|------|---------------------------|----------|------|
| 썸네일 | `thumbnailUrl`(maxres→…→default 폴백) | 호버: `translateY(-4px)`+`glow-card-hover`+썸네일 1.04 | 로딩: stage-800 + shimmer / 에러: 폴백 단계 소진 시 stage-750 + 로고 마크 |
| 뱃지 슬롯(1개) | 우선순위: difficulty 있으면 난이도, 없으면 type | 비인터랙티브 | 없으면 슬롯 비움(빈 박스 금지) |
| 재생시간 칩 | `formattedDuration` | - | 항상 |
| 제목 | `title` | 카드 클릭 → `/watch/[videoId]` | 2줄 클램프 |
| 메타 | `viewCount`+`publishedYear` | - | tabular-nums |

### 3.2 숏츠 카드 (9:16) — `ShortCard` (레일·그리드용, 풀스크린과 별개)

```
┌──────────┐
│       ▶  │  ← 9:16, 썸네일 전면, 우상단 ▶/♪ 라인 아이콘
│          │
│  (썸네일) │
│          │
│●제목오버 │  ← 하단 thumb-veil 그라데이션 위 제목, 좌하단 장르 도트

```

데이터: `thumbnailUrl`, `title`, `genre`(도트 색). 클릭 → `/shorts?v=[videoId]`(피드 진입, 해당 숏츠부터). 이미지 lazy-load + `decoding="async"`.

### 3.3 필터칩 바 — `FilterChips`

```
type:  [전체] [튜토리얼] [워크샵] [퍼포먼스] [직캠] [소셜] [음악] …   ← 1차 축
genre: [● 전체] [● 바차타] [● 살사] [● 주크]                        ← 2차(좌측 6px 도트)
diff:  [전체] [입문] [중급] [고급]   정렬: [인기 ▾]                  ← 보조 + 정렬
```

- 기본칩: 투명 + stage-600 보더 + text-secondary. 활성칩: honey-400 배경 + text-on-honey. 높이 36px.
- 각 칩 우측에 결과 카운트 미니 표기 가능(예: `워크샵 28`). 0건 옵션은 비활성(흐림)으로 표시하되 숨기지 않음(가용성 인지).
- 다중 선택 시 활성칩 우측 ✕(라인 SVG). 상단에 "초기화" ghost 버튼.

### 3.4 콘텐츠 레일 — `ContentRail` (가로 스크롤)

```
섹션 제목                                               [전체 보기 →]
◀ ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐ ▶     (scroll-snap, Rhythm Stagger 진입)
  │카드│ │카드│ │카드│ │카드│ │카드│
  └────┘ └────┘ └────┘ └────┘ └────┘
```

데스크탑: 좌우 ◀▶ 컨트롤 + 마우스 휠/드래그. 모바일: 손가락 스와이프(scroll-snap), 컨트롤 숨김. 우측 카드는 살짝 페이드(더 있음 암시).

### 4.3 구성 요소 표

| 영역 | 데이터 바인딩 | 인터랙션 | 동선 |
|------|--------------|----------|------|
| 히어로 | `channel.json`(title) + 정적 카피 | 모션 배경(Honey Flow + Spotlight) | CTA1→`/learn`, CTA2→`/shorts` |
| 통계 스트립 | `channel.statistics`(19,600·1604·5,237,615 가공) | - | - |
| 트랙 카드(3) | `curated.tracks[]`: title, description, genre, steps.length, 난이도 범위(steps 조인 산출) | 호버 프리뷰, 클릭 | →`/learn/[trackId]` |
| 추천 학습 레일 | ContentCard `isShort=false && type∈{tutorial,workshop}` 인기순 8 | 가로 스크롤, 카드 호버 | 카드→`/watch`, 전체→`/library?type=tutorial` |
| 숏츠 레일 | ContentCard `isShort=true` 인기순/최신 12 | 가로 스크롤 | 카드→`/shorts?v=`, 전체→`/shorts` |
| 장르 칩 | `stats.byGenre`(전체 분포) | 클릭 | →`/library?genre=` 또는 `/shorts?genre=` |

### 4.4 4상태

- **데이터있음:** 위 레이아웃 그대로.
- **빈상태:** 트랙/레일 단위로 처리. 레일 항목 0이면 섹션 자체 비표시(빈 헤더 노출 금지). 트랙은 항상 3개 존재(제거 불가).
- **로딩:** 히어로는 정적이라 즉시. 레일은 카드 스켈레톤 4–5장(stage-800 + shimmer). 통계는 dash(`—`) 후 채움.
- **에러:** 데이터 fetch 실패 시 히어로 + 트랙 카드(정적 가능분)는 유지, 레일은 "콘텐츠를 불러오지 못했어요 [다시 시도]" 인라인 메시지(페이지 전체 죽이지 않음).

완료 기준(spec §3.1): 트랙 링크 동작, 레일 실데이터, 통계 실값, 썸네일 폴백, LCP 모바일 < 2.5s(히어로 모션은 LCP 이후 지연 마운트).

| genre 칩 | 롱폼 byGenre 재계산(도트) | 토글 | bachata 외 희소 — 카운트 노출로 기대치 관리 |
| difficulty 칩 | difficulty≠null 항목만(85건) | 토글 | 활성 시 "난이도 표기 영상만" 안내 |
| 미분류 토글 | unclassified(31 롱폼) | 기본 OFF(숨김) → ON 노출 | 기본 제외(spec) |
| 검색 | title 부분일치(클라) | 입력 디바운스 | 빈 입력=전체 |
| 정렬 | viewCount/publishedAt/duration | 드롭다운 | 기본 인기 |
| 결과 카운트 | 현재 필터 결과 length | 실시간 갱신 | 필터 조합 정확(QA) |
| 카드 그리드 | 필터된 ContentCard[] | 카드→`/watch/[videoId]` | 더보기/무한스크롤 |

### 8.4 인터랙션 플로우 — 필터 적용

```
초기: type=전체, genre=전체, diff=전체, 미분류=OFF, 정렬=인기
  │
  ├─ 칩 클릭 → 즉시 클라 필터 → 카운트·그리드 갱신(Rhythm Stagger 재진입 X, 부드러운 교체)
  ├─ 다중축 AND 결합 (type AND genre AND difficulty AND 검색)
  ├─ URL 동기화: /library?type=workshop&genre=bachata&sort=popular (공유·딥링크)
  └─ 0건 → 빈상태(§8.5)
```

### 8.5 4상태

- **데이터있음:** 그리드 + 카운트.
- **빈상태:** 필터 결과 0 → "이 조건에 맞는 영상이 없어요. [필터 초기화] 또는 다른 조건을 시도하세요." + 가장 가까운 완화 제안(예: "난이도 해제 시 12편"). difficulty 단독으로 빈 결과 양산 방지(spec 완료기준).
- **로딩:** 데이터는 정적 1회 fetch → 첫 로드만 카드 스켈레톤 8장. 필터는 클라라 즉시.
- **에러:** fetch 실패 → "영상 목록을 불러오지 못했어요 [다시 시도]" 그리드 영역 대체.

완료 기준(spec §3.4): type 실분포 반영, difficulty는 ≠null만, 미분류 기본 숨김+토글, 카운트 정확, 더보기/무한스크롤 택1.

| 요소 | 데이터 바인딩 (ContentCard) | 인터랙션 | 상태 |

| 곡 정보 | `song`(29건만) | - | **조건부**(없으면 미표시) |

### 9.4 동선

- 진입: 어디서든 카드 클릭 → `/watch/[videoId]`. 트랙에서는 `?track=`로 컨텍스트 유지.
- 이탈: 관련 영상 → 다른 `/watch`. 트랙 다음 스텝 → 순차. 뒤로 → 직전 목록.
- 숏츠↔상세: 숏츠 "관련 영상 보기"(relatedIds 있을 때) → `/watch`. 역방향 `/watch`에서 숏츠 진입은 relatedIds가 숏츠일 때 동일 레일.

### 9.5 4상태

- **데이터있음:** 플레이어 + 메타 + (조건부)관련/곡/트랙.
- **빈상태:** 관련·곡·트랙 각각 조건부 — 데이터 없으면 **섹션 자체 비표시**(빈 헤더·빈 레일 절대 금지, spec 완료기준). 설명 없으면 설명 블록 생략.
- **로딩:** 플레이어 영역 16:9 스켈레톤 + 제목/메타 라인 스켈레톤.
- **에러:** 존재하지 않는 videoId → **404**(§11). embeddable=false → 인라인 "YouTube에서 보기" 버튼 + 썸네일.

완료 기준(spec §3.6): 임베드 재생, 404, relatedIds/song 있을 때만, 트랙 이전/다음 동작, 썸네일·메타 폴백.

## 13. 구현자 인계 메모

- **ui-craftsman:** 공용 컴포넌트(§3) 먼저 구축 → 페이지 조립. 그리드 분기점 §1 표 준수. 모든 데이터 영역에 §4–10의 4상태 구현(빈 헤더·빈 레일 금지). 토큰은 design-system 스킬 값 그대로.
- **player-integrator:** §7(숏츠 풀스크린·동시임베드≤2·가상스크롤)과 §9(전용 플레이어·트랙 컨텍스트·타임코드 seek·embeddable 폴백) 담당. 숏츠 성능 계약(§7.4)은 frontend-lead 검토 후 확정.
- **copy-writer:** 각 영역 카피 슬롯 — 히어로 헤드/서브(§4), 트랙 설명(§5/6은 curated 원본), 빈상태 메시지(§8.5/11), 404(§11.1), 섹션 제목. 이모지 금지(라인 아이콘), Lorem ipsum 금지.
- **integration-qa:** 라이브러리 카운트 롱폼 부분집합 재계산(§8 헤더), 조건부 렌더(관련·곡·트랙) 빈 섹션 0, 썸네일 폴백 깨짐 0, difficulty 빈결과 방지, 404 동작.

### frontend-lead 회부 (협력성 — 기술 난도 검토)
1. 숏츠 1425건 가상 스크롤 + 동시 IFrame ≤2 + 60fps 스와이프 (§7.4).
2. `/watch` 모바일 스크롤 시 미니플레이어(후순위) 채택 여부.
3. 라이브러리 URL 쿼리 동기화 방식(클라 라우팅 vs 정적).

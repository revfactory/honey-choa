## §A. 톤 가이드 (모든 문구의 기준)

### A.1 브랜드 보이스 — 채널에서 발굴

채널 소개 원문: `놀이터🌱실험실🍯꿀통 / 살사 바차타 주크 라틴댄스`. 꿀초아tv는 가르치는 선생이 아니라 **같이 노는 사람**의 톤이다. "놀이터·실험실"의 가벼움 + 꿀(채널 고유 자산)의 따뜻함이 보이스의 뿌리다.

| 축 | 방향 | 예시 |
|----|------|------|
| **따뜻함 > 권위** | 명령("하세요")보다 동행("같이 해봐요", "따라와요") | ❌ "기초를 학습하십시오" → ⭕ "기초부터 천천히 따라와요" |
| **리듬감** | 짧고 박자 있는 문장. 라틴 음악처럼 끊고 맺기 | "순서대로. 한 스텝씩." |
| **입문자 안심** | 망설임("나도 출 수 있을까")에 먼저 말 건다 | "처음이어도 괜찮아요. 1번부터 시작해요." |
| **댄스 용어 정확** | 바차타/살사/소셜/워크샵/뮤지컬리티 등은 댄서가 쓰는 그대로 | "워크샵 패턴", "소셜 데뷔", "풋워크" |
| **과한 영어 금지** | 정착된 댄스 외래어만 허용. 일반어는 한국어 | ⭕ 소셜·워크샵·풋워크 / ❌ "러닝 패스", "익스플로어" |

### A.2 호칭·인칭
- 사용자 호칭: 직접 호명하지 않음(2인칭 생략). 권할 때만 "~해보세요 / ~해요" 부드러운 청유.
- 채널: 항상 **꿀초아tv** (붙여 씀, tv 소문자). 핸들 **@youzin**.

### A.3 종결어미 규칙
- **본문·안내·빈 상태:** "~해요 / ~어요" (해요체, 따뜻함). 예: "콘텐츠를 불러오지 못했어요"
- **버튼·라벨·뱃지:** 명사형 또는 동사 어간 ("학습 시작하기", "더 보기", "다시 시도"). 종결어미 없이 간결.
- **헤드라인:** 명사 종결 또는 "~다" 단정형 가능(과감함). 예: "라틴댄스를 순서대로 배우다"

### A.4 표기 통일
- 숫자 단위: 만/천 한글 축약(예: `1.2만`, `523만`). 구독자·영상수는 `19.6K`가 아닌 **`1.96만`** 권장하나, 와이어프레임이 `19.6K`를 명시 → **`구독 1.96만`** 으로 한글 통일(아래 푸터/통계 참조).
- 가운뎃점 `·` 으로 메타 구분(쉼표 대신). 예: `조회 1.2만 · 2026`
- 금지: 이모지(라인 아이콘으로 대체), Lorem ipsum, 느낌표 남발(섹션당 최대 1회).

### A.5 글자 수 가드(슬롯 미정 시 기본값)
헤드라인 ≤20자 / 서브카피 ≤40자(2줄 허용) / 섹션 제목 ≤14자 / 버튼 ≤8자 / 칩·뱃지 ≤5자 / 빈상태 본문 ≤45자.

## §B. enum → 한국어 라벨 사전 (ui-craftsman 필수 참조) ★

## §12. 구현·검수 인계 메모

- **ui-craftsman:** enum 라벨은 §B에서만 가져온다(코드에 한국어 하드코딩 금지, §B를 상수 맵으로). 빈/에러 메시지는 §10.3 단일 출처. 트랙 카드 난이도 범위는 §B.5 산출 규칙(null 스텝 제외) — 와이어프레임 placeholder 수치보다 본 덱 우선.
- **player-integrator:** §8.3 임베드 폴백 문구, §7.1 숏츠 상태 문구. 트랙 마지막 스텝 다음 버튼은 "트랙 완주 → 허브로"(§8.1).
- **integration-qa:** 검수 포인트 — (1) 화면에 raw enum(`vlog_etc`, `music_mix` 등) 노출 0건 (2) 빈 헤더·빈 레일 0건 (3) 모든 에러에 복구 버튼 존재 (4) 종결어미 해요체 일관(본문/안내) (5) 이모지 노출은 §9 채널 설명 1곳만 허용.
- **재호출 정책:** 톤 가이드(§A)·enum 사전(§B)을 유지하며 증분 수정. 톤 자체 변경 시 §A 갱신 + 영향 문구 전체 목록과 함께 제안.

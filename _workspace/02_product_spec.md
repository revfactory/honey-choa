## 0. 핵심 데이터 인사이트 (IA의 근거) ★ 먼저 읽을 것

명세에 들어가기 전, IA·우선순위를 결정한 실측 분포다. **가정이 아니라 데이터가 구조를 정한다.**

| 사실 | 수치 | 설계 함의 |
|------|------|----------|
| **숏츠가 압도적 다수** | 1425 / 1604 = **89%** | 숏츠 세로 피드는 보조 페이지가 아니라 **1차 내비게이션 기둥**이다. |
| **롱폼은 희소·고가치** | 179건, 그중 학습형(tutorial+workshop+basics) **85건** | "진짜 강의 카탈로그"는 작고 귀하다 → 라이브러리/커리큘럼의 핵심 자산으로 전면 배치. |
| **장르 쏠림** | bachata 1269(79%), salsa 147, zouk 30 | 장르 단독 필터는 변별력 약함 → **type·difficulty가 1차 필터**, 장르는 보조 필터/탭. |
| **난이도 태깅 희소** | difficulty 보유 477건(int 403·beg 38·adv 36), 나머지 1127건 null | 난이도 필터는 학습형 콘텐츠 한정. 전체 라이브러리 1차 필터로 쓰면 빈 결과 양산 → 라이브러리에선 보조 필터. |
| **추천 데이터 빈약** | relatedIds 보유 28건, song 보유 29건 | 영상↔숏츠 연관 추천·"이 곡으로 배우기"는 **MVP 제외, 후순위**. 있는 28건만 상세에서 노출. |
| **subGenre 멀티** | 138건 | 상세/필터에서 보조 태그로 노출 가능. |
| **커리큘럼 트랙** | 3개(bachata-starter 13step, salsa-starter 10, bachata-social-ready 7) = 30스텝 | 커리큘럼 페이지는 즉시 구축 가능. zouk/kizomba 트랙은 **콘텐츠 부족으로 생성 금지**(아래 §6). |
| **미분류** | unclassified 143(숏츠 112·롱폼 31) | 별도 "미분류" 페이지 만들지 말 것. 라이브러리에 노출하되 type 필터 기본 제외. |

> **선제 경고 (주도성):** 입력 브리프의 type 분포(fancam 406 등)는 숏츠+롱폼 합산치다. 롱폼만 보면 fancam은 17건뿐이고 다수가 숏츠다. 따라서 **"영상 라이브러리"와 "숏츠 피드"는 분리된 데이터 뷰**여야 한다. 합쳐서 한 그리드에 쏟으면 롱폼 강의가 숏츠 직캠 더미에 묻힌다.

## 1. 사이트 목표 · 타깃 사용자

### 1.1 목표
큐레이션된 1604건의 라틴댄스 콘텐츠를 **"보고 끝나는 영상 모음"이 아니라 "단계적으로 배우는 학습 동선"으로** 변환한다. 정적 사이트로 가능한 범위에서 (1) 체계적 학습 경로(트랙), (2) 탐색 가능한 라이브러리, (3) 몰입형 숏츠 피드 — 세 가지 진입로를 제공한다.

### 1.2 타깃 페르소나

| 페르소나 | 동기 | 진입 콘텐츠 | 핵심 니즈 |
|----------|------|------------|----------|
| **P1 입문자 "지원"** (댄스 처음, 어디서 시작할지 모름) | "바차타 배우고 싶은데 순서를 모르겠다" | 커리큘럼 트랙 | 시작점·다음 스텝이 명확한 **가이드된 경로**. 난이도 표시 필수. |
| **P2 취미 댄서 "민지"** (학원 다님, 특정 패턴/곡 복습) | "그 워크샵 패턴 다시 보고싶다 / 이 곡 안무 찾자" | 영상 라이브러리(필터·검색) | type·장르·난이도로 **빠르게 좁혀 찾기**. 거울모드/느리게 튜토리얼. |
| **P3 소셜 댄서 "현우"** (소셜 자주 감, 실전 감각) | "소셜 데뷔 준비 / 직캠·소셜 영상 구경" | social/fancam, social-ready 트랙 | 실전 분위기 콘텐츠, 소셜 준비 트랙. |
| **P4 숏츠 시청자 "유나"** (가볍게 둘러봄, 모바일) | "릴스처럼 댄스 숏츠 넘겨보기" | 숏츠 세로 피드 | **무한 세로 스와이프**, 가벼움. 마음에 들면 채널/관련 롱폼으로 유입. |

> 페르소나 가중치: 데이터상 P4(숏츠) 노출 표면이 가장 크고, P1·P2(학습)가 사이트의 **차별화 가치**다. 디자인은 P4를 입구로, P1·P2를 깊이로 설계한다.

## 2. 정보 구조 (IA) — 페이지/섹션 트리

```
honey-choa (정적 사이트)
│
├── / ……………………………… 홈 (랜딩)
│     ├─ 히어로 (채널 정체성 + 모션 배경)
│     ├─ 커리큘럼 트랙 하이라이트 (3트랙 카드)
│     ├─ 추천 학습 영상 레일 (롱폼 tutorial/workshop, 인기순)
│     ├─ 숏츠 미리보기 레일 (가로 스크롤 → 숏츠 피드 유입)
│     ├─ 장르별 둘러보기 (bachata/salsa/zouk 진입 칩)
│     └─ 채널 통계 스트립 (구독 19.6K · 영상 1604 · 조회 523만)
│
├── /learn ………………………… 커리큘럼 / 학습 트랙 (허브)
│     └── /learn/[trackId] …… 트랙 상세 (순서 있는 스텝 리스트)
│              · bachata-starter / salsa-starter / bachata-social-ready
│
├── /library …………………… 영상 라이브러리 (롱폼 179건, 필터·검색)
│              필터축: ① type ② genre ③ difficulty ④ 정렬(인기/최신/길이)
│
├── /shorts ……………………… 숏츠 세로 피드 (1425건, 풀스크린 스와이프) ★차별화
│              (옵션) /shorts/[videoId] 딥링크 진입
│
├── /watch/[videoId] ……… 영상 상세 / 플레이어 (롱폼·숏츠 공용)
│              · 임베드 플레이어 + 메타 + 관련(relatedIds 보유 시) + 트랙 소속 표시
│
├── /about ……………………… 채널 소개 (꿀초아tv / @youzin)
│
└── /search (MVP는 /library 내장 검색으로 대체 가능) … 통합 검색 (후순위 분리)
```

### 2.1 글로벌 내비게이션 (헤더)
`홈 · 학습(트랙) · 라이브러리 · 숏츠 · 소개` + (후순위) 검색 아이콘.
모바일: 하단 탭바 권장 — `홈 / 학습 / 라이브러리 / 숏츠`. 숏츠는 P4 진입 표면이므로 탭바 1급 항목으로 고정.

### 2.2 라우팅 / 데이터 주입 (frontend 계약)
- 정적 라우트: `/`, `/learn`, `/library`, `/shorts`, `/about`.
- 동적 라우트: `/learn/[trackId]`(3개 사전 생성), `/watch/[videoId]`(1604개 사전 생성 또는 클라이언트 라우팅 + 정적 데이터 조회).
- 모든 페이지는 `_workspace/01_schema.md` §4의 **L3 View Model(ContentCard 등)** 을 빌드타임/런타임에 L1⨝L2 조인하여 소비한다. 정적 JSON은 `/data/*.json`에서 fetch(§01 §0 배치 결정 준수).

## 3. 페이지별 기능 명세 (목적 / 데이터 의존성 / 인터랙션 / 완료 기준)

> 데이터 필드 표기는 `01_schema.md`의 L1(VideoItem)·L2(CuratedItem)·L3(ContentCard/TrackView) 기준.

### 3.1 홈 `/`

| 항목 | 내용 |
|------|------|
| **목적** | 4개 페르소나를 각자의 진입로로 분기. 채널 정체성·차별화(모션) 1초 전달. |
| **데이터 의존성** | `channel.json`(title, handle, statistics). 트랙 하이라이트: `curated.tracks[]` 3개(title, description, 첫 step 썸네일). 학습 레일: ContentCard 중 `isShort=false && type∈{tutorial,workshop}` 인기순 상위 8. 숏츠 레일: ContentCard 중 `isShort=true` 인기순/최신 12. 장르 칩: `stats.byGenre`. |
| **인터랙션** | 히어로 모션 배경(interactive-motion). 레일 가로 스크롤. 트랙/영상 카드 호버 → 상세 프리뷰. CTA "학습 시작하기" → `/learn`. |
| **완료 기준** | (1) 3트랙 카드 렌더·`/learn/[trackId]` 링크 동작. (2) 학습/숏츠 레일이 실데이터로 채워짐(플레이스홀더 0). (3) 채널 통계가 channel.json 실값(19,600·1604·5,237,615 가공). (4) 모든 썸네일 폴백 체인 적용(깨짐 0). (5) LCP 모바일 < 2.5s. |

### 3.2 학습 허브 `/learn`

| 항목 | 내용 |
|------|------|
| **목적** | P1·P3에게 "어디서 시작하나"의 답. 3개 트랙을 학습 목표·난이도 범위와 함께 제시. |
| **데이터 의존성** | `curated.tracks[]`: trackId, title, description, genre, steps.length. 각 트랙 난이도 범위·예상 분량은 steps의 videoId를 ContentCard로 조인해 산출(시작 difficulty ~ 끝 difficulty, 총 영상 수). |
| **인터랙션** | 트랙 카드 클릭 → `/learn/[trackId]`. 트랙별 진행률은 **MVP 제외**(로그인/저장 없음). 후순위: localStorage 기반 "본 영상" 체크. |
| **완료 기준** | 3트랙 모두 카드 노출·링크 동작. 트랙당 스텝 수·장르·난이도 범위 표기. 빈 트랙 0. |

### 3.3 트랙 상세 `/learn/[trackId]`

| 항목 | 내용 |
|------|------|
| **목적** | 순서가 보장된 학습 동선 제시. 입문자가 위→아래로 따라가면 됨. |
| **데이터 의존성** | 해당 `Track`: steps[] = `{videoId, note}`. 각 step을 ContentCard로 조인 → 썸네일·title·durationSeconds·difficulty. TrackView(§01 §4) 형태. |
| **인터랙션** | 세로 타임라인/스텝 리스트(번호·note·난이도 뱃지). 스텝 클릭 → `/watch/[videoId]?track=[trackId]`(트랙 컨텍스트 유지). "다음 스텝" 동선 강조. |
| **완료 기준** | steps가 순서대로 렌더. 난이도가 단조 증가로 표시(역행 시 QA 경고). step의 note 노출. 존재하지 않는 trackId는 404. |

### 3.4 영상 라이브러리 `/library`

| 항목 | 내용 |
|------|------|
| **목적** | P2의 "찾기" 충족. 롱폼 179건을 type·genre·difficulty로 좁혀 탐색. |
| **데이터 의존성** | ContentCard 중 **`isShort=false`** (롱폼 179). 필터 옵션 값: `stats.byType`/`byGenre`/`byDifficulty`(롱폼 부분집합으로 재계산 권장 — 합산 stats는 숏츠 포함이라 라이브러리 카운트와 불일치). 카드: thumbnailUrl, title, type, genre, difficulty, formattedDuration, viewCount, publishedYear. |
| **인터랙션** | 다중 필터(type 1차·genre 2차·difficulty 보조). 정렬: 인기(viewCount)/최신(publishedAt)/길이. 클라이언트 사이드 필터(데이터 정적). 결과 카운트 표시. 카드 클릭 → `/watch/[videoId]`. |
| **완료 기준** | (1) type 필터가 롱폼 실분포 반영(tutorial 50·workshop 28·performance 17 등). (2) difficulty 필터는 difficulty≠null 항목만 대상(빈 결과 방지 안내). (3) unclassified는 기본 숨김(토글로 노출). (4) 필터 조합 시 카운트 정확. (5) 200건 이하라 페이지네이션 불필요하나 무한스크롤/더보기 중 택1. |

### 3.5 숏츠 세로 피드 `/shorts` ★ 차별화 기능

| 항목 | 내용 |
|------|------|
| **목적** | P4 진입 표면. 1425건 숏츠를 릴스/틱톡식 풀스크린 세로 스와이프로 소비. 사이트 체류·재미 담당. |
| **데이터 의존성** | ContentCard 중 **`isShort=true`** (1425). videoId(임베드), title, genre, type, viewCount, relatedIds(보유 시 관련 롱폼 유입). 초기 로드는 인기순/최신 N개 후 점진 로드(전체 1425 동시 임베드 금지 — 성능). |
| **인터랙션** | 풀스크린 1뷰=1숏츠. 세로 스와이프/휠/키보드(↑↓)로 다음·이전. 가시 영역만 YouTube 임베드 마운트(나머지 썸네일 플레이스홀더 → 성능). genre/type 필터 칩(옵션). "관련 영상 보기"(relatedIds 보유 시) → `/watch`. 무한 로드. |
| **완료 기준** | (1) 스와이프 전환 60fps 목표·끊김 없음. (2) 동시 활성 임베드 1~2개로 제한(메모리). (3) 딥링크 `/shorts?v=[videoId]` 또는 `/shorts/[videoId]` 진입 시 해당 숏츠부터. (4) 모바일 세로 전체화면 우선, 데스크톱은 중앙 9:16 컬럼. (5) 1425건 전부 도달 가능(점진 로드 누락 0). |

### 3.6 영상 상세 / 플레이어 `/watch/[videoId]`

| 항목 | 내용 |
|------|------|
| **목적** | 단일 영상 시청 + 맥락(트랙 소속·관련·메타) 제공. 롱폼·숏츠 공용. |
| **데이터 의존성** | ContentCard(L1⨝L2): videoId, title, description, durationSeconds, statistics, type, genre, subGenres, difficulty, song(보유 29건), relatedIds(보유 28건). 트랙 소속: tracks[]에서 해당 videoId 포함 트랙 역참조. uncurated 항목도 노출(§01 §4 정책). |
| **인터랙션** | YouTube IFrame 임베드(embeddable 전건 true, false 폴백 방어). 메타 표시(type·genre 뱃지, difficulty, 조회/좋아요). description의 타임코드 노출(다수 영상 보유). relatedIds 있으면 관련 카드 레일. 트랙 소속 시 "이 트랙 이어보기" 배너. `?track=` 쿼리 있으면 트랙 내 이전/다음 네비. |
| **완료 기준** | (1) 임베드 정상 재생. (2) 존재하지 않는 videoId 404. (3) relatedIds/song은 **있을 때만** 렌더(없는 다수에서 빈 섹션 금지). (4) 트랙 컨텍스트 진입 시 이전/다음 스텝 이동 동작. (5) 썸네일/메타 폴백 적용. |

### 3.7 채널 소개 `/about`

| 항목 | 내용 |
|------|------|

## 5. 사용자 여정 (Journey)

### J1 — 입문자(P1)가 학습 트랙 따라가기
홈 히어로 "학습 시작" → `/learn` 트랙 허브 → `bachata-starter` 선택 → `/learn/bachata-starter` 스텝1(beginner) 시청 → `/watch/[v]?track=bachata-starter` → "다음 스텝" → 순차 진행. **이탈 방지 포인트:** 트랙 상세에서 현재 위치·다음이 항상 보일 것.

### J2 — 취미 댄서(P2)가 특정 콘텐츠 탐색
홈/헤더 → `/library` → type=workshop + genre=bachata + 정렬=인기 → 카드 클릭 → `/watch` 시청 → (있으면) 관련 레일 → description 타임코드로 구간 점프. **핵심:** 필터 카운트가 즉시 정확.

### J3 — 숏츠 시청자(P4)가 둘러보기
모바일 탭바 "숏츠" → `/shorts` 풀스크린 → 세로 스와이프 연속 시청 → 마음에 드는 숏츠에서 "관련 영상"/채널 → `/watch` 또는 `/about` 구독. **핵심:** 가벼운 첫 로드, 끊김 없는 스와이프.

### J4 — 소셜 댄서(P3)가 실전 준비
홈 → `/learn/bachata-social-ready` → 소셜 동선 학습 → `/library` type=social/fancam로 실전 분위기 확인.

## 6. 선제 제안 · 충돌 판정 · 에러 처리

### 6.1 선제 제안 (주도성 90)
- **zouk/kizomba 트랙 생성 금지.** zouk 30건·kizomba 4건으로 5건+단조증가 트랙 구성이 빈약하다. 별도 트랙 페이지를 만들면 빈 깡통이 된다. → 라이브러리 genre 필터로만 노출. (트랙 추가는 콘텐츠 보강 후 재검토.)
- **"미분류" 전용 페이지 만들지 말 것.** 143건은 라이브러리에서 기본 숨김 + 토글 노출로 처리. 별도 IA 노드 불필요.
- **라이브러리·숏츠 데이터 뷰 분리 필수.** §0 경고대로 한 그리드 통합 금지.
- **stats 카운트 이중 계산.** 합산 `stats`는 숏츠 포함이라 라이브러리(롱폼) 필터 카운트와 불일치한다. frontend는 라이브러리 필터 옵션 카운트를 `isShort=false` 부분집합에서 재계산할 것. → integration-qa 검증 항목.

### 6.2 직군 충돌 판정 기준
의견 충돌 시 **사용자 여정(§5)** 을 판정 기준으로 삼고 근거를 본 문서에 기록한다. 예: "숏츠 피드 풀스크린 vs 그리드" 충돌 → J3(P4 몰입·모바일) 기준 풀스크린 채택.

### 6.3 에러 처리
- 콘텐츠 데이터 미존재 시 가정 설계 금지 — 현재는 1604건 실데이터 확인됨, 해당 없음.
- curated에 있으나 RAW에 없는 videoId: §01 §4대로 드롭+경고. 본 명세는 이를 전제.
- 페이지의 데이터 의존성이 빈약(추천·song)하면 해당 섹션을 **조건부 렌더**로 명시(빈 섹션 노출 금지).
